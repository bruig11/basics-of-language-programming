Niveles de Abstracción

¿Qué es la abstracción en programación?

Es la capacidad del código para simplificar conceptos abstractos del mundo real, en otras palabras, permite manejar ideas complejas de forma simple y a veces fragmentadas esto significa que el código puede hacer que la máquina comprenda mejor el mundo real. Un ejemplo de esto se encuentra en la programación Orientada a Objetos. POO funciona encapsulando datos en unidades llamadas clases, estas clases (que se pueden crear en Java y otros lenguajes) permiten, mediante constructores, variables, y el uso de getters y setters, representar y manipular propiedades de un concepto de la vida real y sus cualidades. Así se pueden establecer las características y funciones que debería tener un objeto, facilitando su uso en la programación. A partir de esta idea, podemos decir que existen lenguajes con diferentes grados de abstracción

Lenguajes de Bajo Nivel

¿Qué son?

Son aquellos que interactúan directamente con el hardware. Esto los convierte en una categoría que requiere gran atención al detalle.

Dentro de los lenguajes de bajo nivel se encuentran:

  • Binario: El sistema numérico base 2, compuesto por 0s y 1s, que es el lenguaje más básico que la computadora puede entender directamente ya que cada 1 y 0 significa la presecia de voltaje(Encendido 1) o la asusencia de voltaje (apagado 0).
  • Lenguaje de máquina: Es el lenguaje más básico, compuesto por instrucciones binarias (0s y 1s) que la computadora puede entender directamente. Es específico para cada tipo de procesador.
  • Lenguaje ensamblador: Usa mnemónicos (palabras o abreviaturas) para representar, de forma simbólica, instrucciones que luego son traducidas a binario para que el procesador las ejecute.

¿Cómo interactúan los lenguajes de bajo nivel con el hardware?

Al ser patrones binarios, la Unidad de Control (CU) dentro de la CPU interpreta cada instrucción. Cada vez que recibe un 1 o un 0, se activan o desactivan circuitos eléctricos internos, como las puertas lógicas de los transistores esto provoca que el hardware realice tareas específicas, por ejemplo, leer la memoria RAM.

Usos de los lenguajes de bajo nivel

  • DriversSon programas que le indican al sistema operativo cómo comunicarse con el hardware.
  • Sistemas Operativos (OS)Actúan como traductores entre el usuario, los programas y el hardware.
  • FirmwareEs el software que controla el comportamiento de los dispositivos hardware, como los BIOS o UEFI.

Ejemplo

Bios

El El Basic Input/Output System es una secuencia de código de ejecución almacenada en un chip de la placa base. Su función es reconocer lo que está conectado a la placa base y permitir que el sistema arranque.

¿Cómo lo hace?

El BIOS envía pequeñas corrientes eléctricas a las ranuras y conectores de la placa base. Esto le permite detectar la presencia física de los componentes, verificar que los voltajes sean correctos y obtener información sobre sus características y capacidades para asi confirma los parametros necesarios y asigna los recursos. Siendo un ejemplo muy claro de uso de lenguajes de bajo nivel

Lenguajes de Alto Nivel

¿Qué son?

Son aquellos más compresibles para el ser humano, son compiladores que toman las instrucciones que creas y las transforma a codigo maquina o sea 1 y 0 para que pueda ser interpretado por la maquina

Los compiladores son instrucciones que se traducen a codigo maquina

Usos y Ejemplos

Los lenguajes de alto nivel se utilizan para desarrollar una amplia variedad de aplicaciones, desde software de escritorio hasta aplicaciones web y móviles. Algunos ejemplos populares incluyen:

  • Python: Conocido por su simplicidad
  • Java: Ampliamente utilizado en aplicaciones empresariales y desarrollo de Android.

Detalles sobre la compilación

En el proceso que toma la compilación hay varios pasos que realiza internamente el compilador

  • Análisis léxico: Es un proceso donde se analiza la sintaxis del lenguaje, para así evitar errores y las instrucciones estén estandarizadas
  • Análisis sintáctico: El compilador divide el codigo en tokens /minifaragmentos ejemplo divide por bloques cada if o else y con estos tokens arma un albol sematico donde cada nodo es una operación

    • ¿Qué es un árbol semántico?

    Es una estructura jerárquica que representa las relaciones entre conceptos

  • Análisis semántico: Empieza a pensar a analizar la información de cada bloque construido y a entender el significado de cada operación
  • Optimización: Al haber finalizado la sintaxis y revisar que hace el código se encarga de buscar formas más fáciles de ejecutarlo y reducir al cantidad de memoria utilizada.
  • Generación de código: Se produce el código objeto que puede ser ejecutado por la máquina.

¿Qué son los tipados?

Son el tipo de sintaxis de código que usas para definir un tipo de dato, lo que seria definir una variable y asignarle un valor

Dentos de los compiladores hay tipos de tipados

  • Tipado estático: Siempre tienen un tipo de tipado, sea débil o fuerte, o sea aquellos que no permiten ningún dato a la interpretación

    • Ejemplos: Java, C++, C#, Go

  • Tipado dinámico:Son aquellos que no tienen tipado o tienen un tipado débil, aquel que te permite que las variables no sean definidas con el valor que se va a almacenar dentro del si no que lo intuye automáticamente dependiendo de la variable a definir.

    • Ejemplos: Python, JavaScript, Ruby
